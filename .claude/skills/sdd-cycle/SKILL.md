# sdd-cycle — 기능 한 사이클(스펙→생성→검증→보완) 표준 절차

목적: 이 레포에서 기능을 구현·검증할 때 **"코드 직접 수정 금지, 스펙 대비 검증"** SDD 루프를 표준화한다(Harness의 **Verify·Correct** 축). 규약은 [/CLAUDE.md](../../../CLAUDE.md) 참조.

## 언제 사용
- `lunch-recommender`에서 새 기능/변경을 구현할 때, 또는 생성물을 검증할 때.

## 절차
1. **SPEC 확인** — `docs/spec.md`(무엇)·`docs/plan.md`(어떻게)에서 대상 기능의 요구·성공기준·설계를 읽는다. 요구가 모호/누락이면 **먼저 문서를 보완**(문서 우선).
2. **GENERATE** — `plan.md` 설계대로 코드를 생성한다(바닐라 JS, `config.js` 규약 준수, 정적·무료·키안전 유지).
3. **VERIFY (3층 오라클 대비)** — 생성물을 [`docs/oracle.md`](../../../docs/oracle.md)의 세 오라클과 대조한다(사람 눈 판정 금지). 최소 점검:
   - **① 명세 오라클** (`spec §6` AC + `plan 테스트 관점`) — 후보가 **도보 15분(≈1km) 이내**·**야간 업종 제외**·**최근 N회 안 겹침**·**후보 0 UX**·**메뉴 힌트 도출(추정 표기)**·**거리 직선/도보근사 정직표기**·**지오코딩 실패 안내**·**45개 상한 격자 완화**.
   - **② 도메인 오라클** (`oracle.md` D1~D7 불변식) — AC에 없는 경로에서도 항상 성립하는지: 반경 밖 추천 없음(D1)·야간/비음식 없음(D2)·최근 중복 없음(D3)·**창작 금지**(D4)·**localStorage만·네트워크 유출 없음**(D5)·**JS 앱키만·시크릿 미노출**(D6)·후보 0 크래시 없음(D7). 고정 시드/모의 응답에 대량·랜덤 입력을 넣어 위반 산출이 나오는지 본다.
   - **③ 바이너리 오라클** (golden regression) — 동작 불변 변경(리팩터·성능)이면 `oracle.md`의 golden 스냅샷과 **diff**해 같음을 확인(golden 미존재면 콜드 스타트 — 수용 시 박제).
4. **CORRECT** — 불일치가 있으면 **코드를 직접 고치지 말고**(= Delta를 문서에 반영 후 재생성):
   - 요구 자체가 모호/누락 → `docs/spec.md` 보완 후 2로 복귀(재생성).
   - 설계만 어긋남 → `docs/plan.md` 보완 후 재생성.
   - **도메인 불변식이 스펙에 없어 못 잡음** → `docs/oracle.md`의 도메인 규칙(D-목록)을 보완.
   - 동작을 **의도적으로** 바꾸는 경우 → golden을 손으로 고치지 말고 spec/규칙을 고쳐 재생성 → **golden 재확정(승격)**.
5. **RECORD** — 요청 프롬프트는 `PROMPTS.md`에 누적. 커밋 트레일러는 CLAUDE.md 규칙 준수.

## 금지
- 스펙에 없는 동작을 코드에서 임의 추가/수정(스펙 없는 변경 금지).
- 생성물을 스펙 대비 없이 "눈으로 OK" 판정.
- 시크릿(REST 키 등)·사내 기밀을 코드/문서/커밋에 노출.
