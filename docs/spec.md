# Specification — 점심 추천 (WHAT / WHY)

> 기술(HOW)은 여기 쓰지 않는다. 설계는 [plan.md](plan.md) 참조.

## 1. 문제 정의 (Why)
매일 반복되는 "점심 뭐 먹지" 결정 피로. 회사 주변 선택지는 많지만 늘 가던 곳만 가거나 고르기 귀찮다. **한 번의 클릭으로 회사 주변에서 랜덤하게, 단 최근에 추천된 곳은 빼고** 한 곳을 정해주면 결정 비용이 사라진다.

## 2. 사용자 (Who)
회사(특정 사옥) 근처에서 일하는 개인. 로그인 없이 즉시 사용. 주 사용 시간대는 평일 11:30~13:00, 모바일.

## 3. 핵심 유즈케이스 (What)
- **UC1 추천받기**: 사용자가 "오늘 점심 추천" 버튼을 누르면, 회사 중심 반경 내 음식점 중 **최근 추천 이력에 없는** 한 곳을 무작위로 골라 보여준다.
- **UC2 다시 추천**: 결과가 마음에 안 들면 "다른 곳" 버튼으로 즉시 다른 한 곳을 다시 뽑는다(방금 것도 제외).
- **UC3 상세로 이동**: 추천된 식당의 이름·업종(카테고리)·거리를 보고, "카카오맵에서 보기"로 위치·메뉴·리뷰를 확인한다.
- **UC4 이력 확인/초기화**: 최근 추천된 목록을 보고, 원하면 이력을 초기화한다(다시 전체가 후보가 됨).

## 4. 기능 요구 (What, not How)
- **회사 위치**: `서울특별시 성동구 아차산로13길 11` 을 중심으로 한다.
- **거리 제한**: 회사로부터 **도보 15분 이내** 식당만 후보로 한다. (도보 15분 ≈ 직선 약 1,000m 로 근사 — 근거·조정은 [plan.md](plan.md))
- **점심 영업 제약**: **점심 시간대에 영업하는 식당만** 후보로 한다.
  - ⚠️ 한계: Kakao 검색 API는 **영업시간을 제공하지 않아** "점심 영업 중"을 데이터로 보장할 수 없다. 따라서 아래로 **근사**한다(§8 참조):
    - (자동) 음식점(FD6)만 대상, **카페(CE7) 제외**, 카테고리명이 **야간 전용 업종**(술집/호프/바/포차/포장마차/야식 등)이면 제외.
    - (선택·정확) 실제 점심 영업 여부를 수동 확인한 **제외/포함 오버라이드** 목록으로 보정.
  - 최종 영업여부는 표시하지 않고(창작 금지) **카카오맵 링크로 위임**한다.
- 위 조건을 통과한 후보 중 **최근 N개(기본 예: 10) 추천 이력을 제외**하고 무작위 1곳 선택.
- 후보가 이력으로 모두 소진되면 가장 오래된 것부터 다시 후보에 포함(순환).
- 결과에 **식당명, 업종/카테고리, 회사로부터 거리, 지도상 위치, 카카오맵 링크**를 표시.
- 추천 이력은 브라우저에 유지되어 재방문 시에도 안 겹친다.

## 5. 비기능 요구
- 버튼 클릭 → 결과까지 **3초 이내**.
- 모바일 우선 반응형, 다크/라이트 대응.
- 완전 정적·무료 배포. 서버·계정·개인정보 수집 없음.

## 6. 성공 기준 (Done)
- 한 번의 클릭으로 반경 내 **최근과 겹치지 않는** 식당 1곳을 지도·거리·카카오맵 링크와 함께 제시한다.
- 같은 사용자가 연속으로 눌러도 최근 N회는 서로 다른 곳이 나온다.

## 7. 범위 밖 (Non-goals, 현재)
- 팀 투표·공유 이력(백엔드 필요) — 개인용 원칙상 제외.
- 로그인/계정, 서버 저장.
- **실시간 영업여부·정확한 영업시간·상세 메뉴/가격 크롤링** — Kakao 검색 API 미제공. 카카오맵 링크로 위임(§4 점심 영업 제약, §8 참조).

## 8. 미해결 / 제약 (Clarify)
- [x] **회사 위치** — `서울특별시 성동구 아차산로13길 11` 로 확정. (좌표는 Kakao Geocoder로 주소→좌표 변환, [plan.md](plan.md))
- [x] **거리 제한** — 도보 15분 이내로 확정. 직선 약 1,000m 로 근사(우회·보행속도 가정은 plan.md, 조정 가능).
- [ ] **점심 영업 판별의 정확도** — Kakao가 영업시간을 안 주므로 §4의 근사(카테고리 제외 + 수동 오버라이드)로 처리. 정확도가 부족하면 추후 (a) 소수 대상 수동 큐레이션 확대 또는 (b) 영업시간 제공 데이터소스 검토. **자동만으로 100% 보장은 불가**함을 사용자에게 고지.
- [ ] **"메뉴 추천" 범위**: Kakao Local은 개별 메뉴/가격을 제공하지 않는다. 메뉴는 **업종/카테고리(예: 한식>국밥, 일식>돈카츠) 수준**으로 제안하고 상세는 카카오맵 링크로 위임. (직접 메뉴 DB 큐레이션은 추후 결정)
- [ ] 반경 후보가 너무 많을 때 카테고리 필터(한식/양식/매운거 등) 도입 여부(후속).
