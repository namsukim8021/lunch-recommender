# Constitution — 프로젝트 불변 원칙

SDD의 최상위 문서. 이후 spec/plan/구현은 아래 원칙을 위반할 수 없다.

1. **정적·무료 우선** — GitHub Pages로 배포 가능한 순수 정적(백엔드 없음)을 기본으로 한다. 백엔드가 필요한 기능은 원칙적으로 범위에서 제외한다.
2. **개인정보 최소** — 추천 이력·취향은 브라우저 `localStorage`에만 저장하고 서버로 보내지 않는다. 로그인·계정 없음.
3. **데이터 정확성 (창작 금지)** — 존재하지 않는 식당·메뉴를 지어내지 않는다. 표시하는 식당은 Kakao 실데이터에 근거한다. 근거 없는 값은 표시하지 않는다.
4. **키 안전** — 클라이언트에 노출되는 키는 **도메인 제한된 Kakao JS 앱키만** 사용한다. REST 키 등 노출 시 위험한 시크릿은 정적 코드/레포에 넣지 않는다.
5. **한국어 UI · 모바일 우선** — 점심시간 휴대폰 사용을 1순위로 설계한다.
6. **단순함 → 반복** — 한 번에 완성하지 않는다. 동작하는 최소본(MVP)을 먼저 만들고 반복 개선한다.
7. **명세 우선** — 기능 추가는 먼저 [spec.md](spec.md)에 WHAT/WHY로 기술한 뒤 구현한다.
